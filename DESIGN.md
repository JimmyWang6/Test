## 1. 项目概述

本项目是一款基于纯原生 Web 技术（HTML / CSS / JavaScript）实现的网页版五子棋游戏，无需安装任何依赖，用浏览器打开 `index.html` 即可运行。

游戏支持 **双人本地对战** 和 **人机对战** 两种模式，具备完整的落子记录、悔棋、积分统计、胜负提示等功能，并提供流畅的交互体验和美观的视觉界面。

## 6. 模块设计

### 6.1 棋盘渲染

- 棋盘使用 CSS Grid（`15×15`）生成 225 个 `<div class="cell">` 格子。
- 网格线通过每个 `cell` 的 `::before`（横线）和 `::after`（竖线）伪元素绘制。
- 边缘格子通过 `.edge-top/.edge-bottom/.edge-left/.edge-right` 裁剪伪元素，使网格线不超出棋盘外框。
- 天元和星位（共 9 个）标注 `.star-point` 类，以实心圆点表示。
- 棋子为绝对定位的 `<div class="stone black|white">`，尺寸为格子大小的 82%，带径向渐变模拟立体感。

### 6.2 游戏状态管理

核心状态变量：

| 变量 | 类型 | 说明 |
|------|------|------|
| `board` | `number[][]` | 15×15 二维数组，`0`=空，`1`=黑，`2`=白 |
| `currentTurn` | `number` | 当前行棋方（`BLACK=1` / `WHITE=2`） |
| `gameOver` | `boolean` | 是否已分出胜负 |
| `moveHistory` | `object[]` | 落子顺序列表，每项含 `{row, col, player}` |
| `scores` | `object` | 各方得分，格式 `{1: n, 2: m}` |
| `mode` | `string` | `'two-player'` 或 `'vs-ai'` |
| `aiThinking` | `boolean` | AI 是否正在计算（锁定玩家输入） |

状态变更流程：

```
用户点击格子
    │
    ▼
onCellClick() → placeStone(row, col)
    │
    ├─ 更新 board[][] 和 moveHistory
    ├─ renderBoard() + renderHistory()
    ├─ checkWin() → 胜利？→ 弹出获胜框，锁定游戏
    ├─ 棋盘满？→ 平局提示
    └─ 切换 currentTurn → 若 VS-AI 且轮到 AI → setTimeout(doAIMove)
```

### 6.3 胜负判断

在落子点 `(row, col)` 处，沿以下 4 个方向（每方向双向延伸）统计连续同色棋子数：

| 方向 | 向量 `[dr, dc]` |
|------|-----------------|
| 横向 | `[0, +1]` |
| 纵向 | `[+1, 0]` |
| 正对角线 | `[+1, +1]` |
| 反对角线 | `[+1, -1]` |

任意方向连子数 ≥ 5 即获胜（允许长连）。时间复杂度 O(1)（每次最多扫描 4×14=56 步）。

### 6.4 AI 算法

**算法类型**：威胁评分启发式（Threat-based Heuristic），不使用 Minimax/Alpha-Beta（保证响应速度）。

**候选点生成**（`getCandidates()`）：仅在已有棋子周围 2 格半径内的空格中搜索，大幅减少评估量。

**评分函数**（`scoreDir()`）：对每个方向上的连子数和两端开放情况打分：

| 连子数 | 两端开放 | 分值 |
|--------|----------|------|
| ≥ 5 | - | 100,000（必胜） |
| 4 | 双开 | 10,000（活四） |
| 4 | 单开 | 1,000（冲四） |
| 3 | 双开 | 500（活三） |
| 3 | 单开 | 100（眠三） |
| 2 | 双开 | 50 |
| 2 | 单开 | 10 |

**决策逻辑**（`aiPickMove()`）：
1. 若存在立即获胜的落点 → 直接落子。
2. 否则，对每个候选点计算 **进攻分 × 1.1 + 防守分**，取最高分落子。
3. 同分时随机选取，增加多样性。

**触发时机**：玩家落子后延迟 `AI_MOVE_DELAY_MS`（120 ms）再执行，确保 UI 先完成重绘。

### 6.5 UI 交互

| 交互 | 实现方式 |
|------|----------|
| 悬停预览 | `mouseenter` 动态插入半透明预览棋子 DOM；`mouseleave` 移除 |
| 最新落子标记 | 最后一步棋子追加 `.last-move` 类，CSS 绘制红色圆点 |
| 获胜弹层 | `#win-overlay` 固定定位全屏遮罩，CSS `@keyframes pop` 弹出动画 |
| 模式切换 | 点击模式按钮更新 `.active` 类并调用 `initBoard()` 重置 |
| 悔棋 | 从 `moveHistory` 弹出条目并清除 `board[][]` 对应位置，重新渲染 |
| 落子记录 | 每次落子后增量追加并自动滚动到底部 |

## 7. 数据结构

### 棋盘状态（`board`）

```
board[row][col] → 0 | 1 | 2
  行 row：0（上）~ 14（下）
  列 col：0（左）~ 14（右）
```

### 落子记录（`moveHistory`）

```js
[
  { row: 7, col: 7, player: 1 },  // 第 1 步：黑棋天元
  { row: 7, col: 8, player: 2 },  // 第 2 步：白棋
  ...
]
```

### 坐标记谱转换

- 列：`'ABCDEFGHJKLMNOP'[col]`（标准五子棋跳过字母 I）
- 行：`15 - row`（使棋盘顶部为第 15 行，符合棋谱惯例）
- 示例：`(row=7, col=7)` → `H8`（天元）
